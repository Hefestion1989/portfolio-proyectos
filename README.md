## Mapa del laboratorio

Los proyectos y materiales se organizan desde [`laboratorio-digital`](https://github.com/Hefestion1989/laboratorio-digital):

- `proyectos/`: herramientas, prototipos y apps con potencial de publicacion.
- `experimentos/`: pruebas rapidas e ideas en desarrollo.
- `notas/`: decisiones, contexto y pendientes.
- `archivo/`: materiales editoriales y paquetes que conviene conservar.
- `utilidades/`: scripts y reportes de mantenimiento.

## Proyectos tecnicos

- `VeryCheck`: producto experimental para analizar afirmaciones y devolver una evaluacion estructurada con fuentes cuando esten disponibles.
- `promptmaster-ai`: version mas completa del taller de prompts, con React, Vite y Gemini.
- `Prompts`: version liviana y offline del taller de prompts, usable desde el navegador sin cuenta ni API key.
- `academic-semantic-search`: buscador academico local para documentos.
- `pdf-concept-search`: prototipo de busqueda conceptual sobre PDFs.
- `office-copilot`: macros para usar ChatGPT desde Excel y Word.
- `procesamiento-txt`: flujo simple para mover, limpiar y registrar archivos TXT.

## Archivo editorial

- `subjetividad-no-biologica`: ensayo sobre subjetividad artificial.
- `ensayos-ia-conciencia-sociedad`: coleccion academica sobre IA, conciencia, subjetividad y gobernanza tecnologica.

## Criterio de organizacion

- El portfolio muestra y explica.
- El laboratorio guarda y desarrolla.
- Los proyectos tecnicos viven dentro de `laboratorio-digital/proyectos/`.
- Los materiales editoriales viven dentro de `laboratorio-digital/archivo/editorial/`.
- Los repos individuales se mantienen cuando conviene que una pieza sea visible por si sola.
